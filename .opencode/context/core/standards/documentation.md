# Documentation Standard

## Goals

- Keep docs concise, actionable, and implementation-aligned.
- Update user-facing behavior and operational guidance when workflows change.

## Rules

- Prefer short sections with direct examples.
- Avoid marketing language in technical docs.
- Keep file references accurate and current.

## Required updates

- Any workflow, policy, or command changes
- Any new skill/subagent responsibilities
- Any new context loading requirements
